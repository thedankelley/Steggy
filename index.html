Steggy

Steggy Guidebook

What is Steggy?

Steggy is a client-side application for hiding information inside ordinary-looking images or audio. This technique is called steganography. Unlike typical encrypted messengers, Steggy does not look like encrypted communication.

Who should use Steggy?

  • Journalists working under censorship
  • Human rights researchers
  • Activists in hostile regions
  • Emergency or contingency communication planners

Protected vs Decoy Messages

Steggy allows you to embed two messages: a decoy message that is easy to extract, and a protected message that requires the correct method or key. This provides plausible deniability.

AES Encryption

AES uses a shared password. Anyone with the password can decrypt the message. This is simple but requires secure password exchange.

PGP Encryption

PGP uses public and private keys. Anyone can encrypt a message using your public key, but only you can decrypt it using your private key.

Why use AES and PGP together?

Layering AES and PGP adds defense in depth. If one layer is compromised, the other may still protect the message.

Fragmentation

Fragmentation splits data across multiple images or audio files. This reduces the impact of loss or corruption and can help evade detection.

SSTV (Slow Scan Television)

SSTV converts images into audio signals that can be transmitted over radio, phone lines, or any audio channel. This is useful when internet access is blocked.

Threat Model

Steggy is designed to evade casual inspection and automated scanning. It does NOT protect against advanced forensic analysis by powerful state actors.

Important Warnings

  • No system is perfectly secure
  • Always test before real-world use
  • Assume your device could be compromised

Offline Use

Steggy runs entirely in your browser. Once loaded, it can be used offline. No data is sent to any server.
